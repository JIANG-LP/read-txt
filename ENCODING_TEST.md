## 🛠️ 手动选择编码测试

### 测试步骤

1. **打开命令面板**
   ```
   Ctrl+Shift+P
   ```

2. **输入命令**
   ```
   TXT Reader: 手动选择编码
   ```

3. **选择编码选项**
   ```
   - UTF-8 (通用编码，支持多国语言)
   - GBK (简体中文常用编码)
   - GB2312 (简体中文标准编码)
   - GB18030 (中文扩展编码)
   - BIG5 (繁体中文编码)
   - Latin-1 (西欧语言编码)
   ```

4. **重新选择文件**
   - 选择对应的编码后
   - 重新打开之前显示乱码的文件
   - 应该正常显示

## 🐛 已知限制

### 不支持的编码

以下编码目前不支持（未来可能添加）：

- ❌ UTF-16 LE/BE
- ❌ UTF-32
- ❌ EUC-JP（日文）
- ❌ Shift_JIS（日文）
- ❌ EUC-KR（韩文）

### 特殊情况

1. **损坏的文件**
   - 文件不完整
   - 编码信息丢失
   - 可能无法正确检测

2. **混合编码**
   - 同一文件中使用多种编码
   - 可能导致部分乱码

3. **非常见编码**
   - 特定行业的专用编码
   - 老旧系统的特殊编码
   - 需要手动转换

## 🔧 故障排查

### 问题 1: 仍然显示乱码

**排查步骤：**

1. **查看当前编码**
   - 鼠标悬停在状态栏文本上
   - 查看显示的编码格式

2. **尝试手动指定编码**
   ```
   Ctrl+Shift+P → TXT Reader: 手动选择编码 → 选择 GBK
   ```

3. **查看控制台日志**
   ```
   帮助 → 切换开发人员工具 → 控制台
   查看编码检测相关日志
   ```

4. **转换文件编码**
   ```bash
   # 用记事本另存为 UTF-8
   # 或用 VSCode 转换为 UTF-8
   ```

### 问题 2: 检测到的编码不正确

**解决方法：**

1. **记录文件信息**
   - 文件大小
   - 创建方式
   - 使用的软件

2. **使用工具确认编码**
   - Windows: Notepad++
   - macOS: TextEdit
   - 在线工具：detectencoding.com

3. **提交 Issue**
   - 提供文件样本
   - 说明检测到的编码
   - 说明实际应该的编码

### 问题 3: 性能问题

**优化建议：**

1. **减少自动刷新频率**
   ```json
   {
     "txtReader.refreshInterval": 0  // 关闭自动刷新
   }
   ```

2. **避免超大文件**
   - 单次加载 < 10000 行
   - 分割大文件

3. **关闭不必要的监听**
   - 不修改的文件不需要监听
